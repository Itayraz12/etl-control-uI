STEP 1 – File Upload
---------------------------------------------------

  - Infer schema

- Persist detected fileType for later conditional UI logic

---------------------------------------------------

  productType

- Define input format configuration section:

  - File Encoding (default: UTF-8)
  - Date Default Format
  - Optional format description

  If fileType === JSON:

---------------------------------------------------
STEP 3 – Field Mapping (Visual Canvas + Rules Engine)
---------------------------------------------------

Goal:
Design an advanced mapping experience inspired by a visual node/edge canvas editor.

Core UX Layout:

- Left Panel: Source Fields Tree
  - Hierarchical JSON path viewer
  - Expand/collapse
  - Search
  - Array indicators
  - Type badges
  - Virtualized rendering for large trees

- Right Panel: Target Entity Fields Tree
  - Required markers
  - Type indicators
  - Search
  - Validation state indicators

- Center Panel: Interactive Mapping Canvas

Mapping Canvas Concept:

- Each mapping is represented as a visual edge (connection)
- Source and Target fields are draggable node chips/cards
- Nodes contain:
  - Field name (truncated if needed)
  - Type label
  - Icon
  - Left/right connection ports
- Edges:
  - Smooth curved lines
  - Direction left → right
  - Redraw only when node position changes

User interactions:

- Drag source field onto canvas
- Drag target field onto canvas
- Connect ports (source → target)
- Reposition nodes freely
- Delete edge
- Clear connections per node
- Multi-select delete
- Zoom and pan support
- Snap-to-grid optional

Mapping Rules Panel:

- Side panel listing all mappings:
  - Source Path → Target Field
  - Type match indicator
  - Required coverage indicator
  - Status (valid / warning / error)
  - Quick actions: edit, disable, delete

Right-click Context Menu (Cascading Submenus):

Right-click on edge or node opens cascading menu:

Header:
  sourcePath → targetField

Sections:

1) Array Handling (single choice)
   - Map first element
   - Map each element
   - Flatten
   - Aggregate (SUM / COUNT / etc.)

2) Null Handling (single choice)
   - Fail record
   - Skip record
   - Default value

3) Validation Constraints (multi-select)
   - Required
   - Min/Max
   - Regex
   - Allowed values

4) Notes
   - Free text note
   - Clear note option

Context actions:
- Edit rule (open modal)
- Clear connections
- Delete mapping

Per-instance rule model:

- Each mapping edge has independent rule properties
- Support optional Rule Templates:
  - Define default rule presets
  - New mappings may inherit defaults
  - Edges remain independently editable

Validation logic:

- All required target fields must be mapped before proceeding
- Detect type mismatches
- Suggest transformations when mismatch occurs
- Inline warnings on canvas edges
- Show issues in mapping table
- Support Auto-map (fuzzy matching) with review confirmation

Performance constraints:

- Virtualize large field trees
- Debounce drag operations
- Redraw edges only on node position updates
- Clean up mapping state when nodes/edges removed
- Avoid memory leaks
- Lazy load deep JSON branches
